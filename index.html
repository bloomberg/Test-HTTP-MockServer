Test::HTTP::MockServer
Testing things that don't want to be tested
Daniel Ruoso <druoso@bloomberg.net>

The Problem

  • Your code uses a library that makes HTTP calls
  • But no way to inject a test HTTP client

Example

  • You're trying to test code that calls the GitHub API
  • But the author didn't give you many configuration options

Can we inject it?

The author only allows overriding the url, not the UA

Can we monkey-patch it?

Ugh, yes.

But, how much of it?

What if it changes?

Fake GitHub?

Yeah, but that's a lot of work

Ok, so what then?

Create a server that knows how to mock in the context of this test specifically

Initialization

use MyAutomation;
use Test::HTTP::MockServer;
use MyAutomation;
my $server = Test::HTTP::MockServer->new();
my $url    = $server->url_base();
my $autom  = MyAutomation(
  githuburl => $url,
  org       => "myorg",
  workspace => "t/data/scratch/",
);

Mock for this specific test

use Test::HTTP::MockServer::REST;
my $rest = Test::HTTP::MockServer::REST->new(
  listrepos => qr/repos$/,
)
my $requestprocessor = $rest->wrap_hash(
  listrepos => sub {
    return [ { name => "foo",
               clone_url => "t/data/template/foo.git" },
             { name => "bar",
               clone_url => "t/data/template/bar.git" } ]
  }
);

Test against the MockServer

$server->start_mock_server($requestprocessor);

my $rc = $autom->clone_all_repos();
ok($rc, 'success cloning all repos');
ok(-d "t/data/scratch/foo/.git", "cloned myorg/foo");
ok(-d "t/data/scratch/bar/.git", "cloned myorg/bar");

$server->stop_mock_server($requestprocessor);

Mock for the next test

use Test::HTTP::MockServer::REST;
my $rest = Test::HTTP::MockServer::REST->new(
  issues => qr/issues$/,
)
my $requestprocessor = $rest->wrap_hash(
  issues => sub {
    return [ { number => 1,
               title => "my pr" } ]
  }
);

Test against the MockServer

$server->start_mock_server($requestprocessor);

my @pulls = $autom->list_pull_requests();
is(scalar @pulls, 1, "found the one pull request");
is($pulls[0]{title}, "my pr", "got the right data");

$server->stop_mock_server($requestprocessor);

How does it work?

Binds to a random TCP port

Try again if it's already in use

How does it work?

Fork after the bind on start_mock_server

Kill the server on stop_mock_server

How does it work?

Process one request at a time

Single-thread, single-process

Easy to make things in sequence

How does it work?

Call the mock server yourself in more complex scenarios

Copy-on-write, so no state flows back
